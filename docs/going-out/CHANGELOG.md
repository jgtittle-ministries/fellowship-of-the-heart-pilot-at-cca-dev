# Architectural status

Two parallel architectural designs for the spring volume still coexist in the corpus:

- Design A — Leadership equipping (Discernment / Equipping / Launch). Three phases of four weeks each. Focus: equipping calling-confirmed participants to start their own formation groups in their own rooms. This is the architecture of the new Going Out Handbook (IJH\_GoingOut\_Handbook\_v1.docx).
- Design B — Body sent + outward witness (Body Sent / Mission Close-to-Home / Mission Far / Sending). Four blocks of three weeks each. Focus: the body’s sent identity, daily Tells, witness in household / vocational / third-place contexts, mission discernment in each member’s own life. This is the architecture described in the actual GO weekly session plans (GO\_Wk1\_WelcomeBack.docx through GO\_Wk12\_LongObedience.docx).

The handbook (Design A) and the weekly sessions (Design B) describe different volumes. The team will need to reconcile this before the Going Out pilot runs — either by rewriting the handbook to match the sessions, or by rewriting the sessions to match the handbook, or by clearly framing the handbook as a separate 'Inviting Others' offering that runs after Going Out (which was the user's May 2026 decision and the reason for the prior round's repurposing).

This bundle does not resolve the conflict; it presents both architectures and lets the user decide. The prior round's Inviting Others Handbook (which carries Design A's content with Inviting Others naming) is retained in the bundle as the alternative framing.

# Three Taxonomies card — note on placement

The Three Taxonomies disambiguation card has now been added to both the GD handbook (Appendix C opener) and the new Going Out Handbook (between the Letter and Section 1). The card content is identical across the two locations, adapted only to point readers to the correct local section for the Leader Level taxonomy.

The GS handbook did not receive this card per the prior round's decision (GS readers only encounter the Affective Taxonomy, and the section header sets sufficient context).
